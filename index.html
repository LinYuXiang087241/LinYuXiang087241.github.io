set-up-wireguard

发表于 2022-03-23 分类于 configuration

在centos7配置wireguard

wireguard 服务端配置

    1. 下载必要软件包
    1. 配置内核参数
    1. 配置iptables转发
    1. 生成wg秘钥
    1. 安装docker，docker-compose
    1. 使用docker-compose启动wireguard-web
    • 检查docker0 网桥的地址
    • compose文件中的红色ip改为docker0桥的地址
    • 启动docker-compose
    1. 通过网页对wireguard网关进行配置
    • server interface address 为wireguard网段的网关地址
    • public endpoint为当前wireguard主机的地址| Default allowd ip处填写与1中网关相同的网段
    • 对应图中的内容进行修改
    • 配置完成后，检查对应 /etc/wireguard/wg0.conf 配置文件是否更改
    1. 配置系统自动加载配置文件、自动生成配置文件
    • 加入ExecReload在不中断活跃连接的情况下重新加载配置文件
    • 为wg-gen-web添加自动Reload
    • 添加wg-gen-web监听路径
    • 配置完成后刷新配置

  • 9 加入客户端前的操作

    • 在wireguard网页点击ADD NEW CLIENT创建client
    • Client friendly name为必填项，填写完成点击SUBMIT创建

加入客户端

windows客户端

    1. 登录wireguard.com 下载win专用的客户端
    1. 下载完成双击安装（安装过程需要可以访问外网）
    1. 安装完成后，下载服务端对应client的配置文件，如图点击DOWNLOAD，下载下来的便是xxxx.conf
  • 双击打开安装好的wireguard客户端
  • 点击从文件导入隧道即可建立wiregurad连接

centos客户端

    1. 下载必要软件包
    1. 从服务端下载client端的配置文件xxxx.conf，并将其改名为wg0.conf
    1. 通过服务的方式启动wireguard

安全加固

  • 配置除了内网ip外，其他ip无法访问
  • 重新启动docker-compose

memory-failed-write-dqdqs-DIMM-A2

发表于 2022-03-23 分类于 usage

服务器开机时报出 memeory failed错误，如下图

  • 原因：
    • 如果是第一次打开的服务器，可能是由于对应的内存松动引起，重新插拔内存即可修复。参考此DELL官方回复
    • 如果是旧服务器，可能由于内存功率过大引起。恢复办法有两种：
      • 换更低功率的内存
      • 少插几根内存

    4. 通过下图配置硬盘为raid0直通服务器
    5. 保存配置后，退出重启服务器。

vmware-esxi6.5-reset-password

发表于 2022-03-19 更新于 2022-03-23 分类于 usage

重置vmware esxi6.5 root 密码

  • 准备：

    • RHEL 7 启动U盘或ISO
    • 服务器配置从U盘或ISO启动

  • 详细步骤：

    1. 从U盘启动后，选择Troubleshooting
    2. 选择Rescue a Red Hat Enterprise Linux system
    3. 选择1回车
    4. 挂载sda5设备后，将root密码移除 删除$ 至:18934之前

      删除后如图：
    5. 将修改后的shadow文件打包放回
    6. 退出救援模式，配置从硬盘启动。启动后，登录名为root，密码为空。
